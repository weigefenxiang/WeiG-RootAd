# Contributing / 参与贡献

Use the rule-report Issue form for missed ads, false positives, or reward failures. Include the app package/version, Android version, Root framework, active profile, rule version, exact suspected domains, timestamps, and the result of allowing or blocking one domain.

Do not upload cookies, account identifiers, authorization headers, request bodies, full private HTTPS URLs, device serial numbers, or unrelated captures. The manager's one-tap Issue button pre-fills safe diagnostics only.

Add maintained overrides to `rules/local/` or a reviewed upstream to `rules/sources.json`. Do not hand-edit `rules/generated/`. Every external list needs a stable source URL and a compatible license.

漏拦、误杀或奖励广告失败请使用规则 Issue 模板，提供应用包名/版本、Android、Root 方案、模式、规则版本、精确可疑域名、时间以及单独放行/拦截后的结果。禁止提交 Cookie、账号信息、授权头、请求正文、私人完整 URL、设备序列号或无关抓包。
